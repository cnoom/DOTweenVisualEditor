# 📋 项目文档：DOTween 可视化编辑器（优化版）

## 1. 项目概述

- **项目名称**：DOTween 可视化编辑器
- **目标**：为 Unity 开发者创建一个开源、高性能、易扩展的编辑器扩展，提供媲美 DOTween Pro 的可视化动画编辑体验，并克服现有同类开源项目（如 Animation Sequencer）的弊端。

### 核心差异化设计

1.  **数据与对象彻底解耦**：动画序列可复用、可合并。
2.  **轨道/层级时间轴界面**：轻松管理数百个动画步骤。
3.  **运行时与编辑时行为严格一致**：提供可视化调试工具。
4.  **API 兼容适配**：支持 DOTween 基础版与专业版双 API 适配。
5.  **轻量级扩展模型**：非程序员也可通过配置文件添加新动画类型。

## 2. 需要规避或优化的主要弊端（来自 Animation Sequencer 分析）

| 弊端类别 | 具体问题 | 本项目的解决方案 |
| :--- | :--- | :--- |
| **依赖绑定** | 硬编码依赖 DOTween，且需要用户手动配置 asmdef | 使用条件编译 + 运行时检测，自动兼容 Free/Pro；提供一键式 asmdef 设置向导 |
| **架构设计** | 序列化结构复杂、耦合 Unity 序列化系统；编辑时与运行时行为不一致 | 采用纯数据模型 + 独立播放器层；预览和播放使用完全相同的执行路径，消除差异 |
| **功能覆盖** | 内置动画类型少，路径、曲线控制弱 | 预置 20+ 常用动画步骤（包括路径、粒子、材质、TextMeshPro 等），支持自定义曲线导入 |
| **性能表现** | 预创建所有 Tween 导致固定内存开销；GC 压力较大 | 改为懒加载 + 对象池模式，仅在播放时创建 Tween，播放完立即回收；零 GC 关键路径 |
| **用户体验** | 大型序列管理不便，缺乏轨道/层级；调试工具薄弱 | 实现轨道（Track）和片段（Clip）时间轴 UI，支持折叠、静音、独奏；提供逐步骤高亮、属性曲线监视器 |
| **数据存储** | 数据与 GameObject 强绑定，无法复用，版本控制冲突多 | 动画序列保存为独立 ScriptableObject 资产，支持多个对象引用同一序列，合并友好 |
| **扩展性** | 自定义动画动作需要写代码，门槛高 | 提供可视化脚本图形（GraphView）编辑器，用户可通过拖拽节点创建新动画类型，生成对应的步骤代码 |
| **活跃度** | 原项目维护不活跃 | 本项目采用模块化架构，核心 API 稳定，鼓励社区贡献；计划每季度发布一次功能更新 |

## 3. 技术架构设计（优化版）

### 3.1 整体分层

```text
[编辑器层] UI Toolkit 窗口 + GraphView 扩展节点编辑器
     ↓
[序列化层] ScriptableObject 存储纯数据模型（无 MonoBehaviour 依赖）
     ↓
[播放器层] AnimationPlayer (MonoBehaviour) 引用数据模型，运行时执行
     ↓
[适配器层] DOTweenAdapter (封装 DOTween API，支持 Free/Pro 切换)
```

### 3.2 核心模块

#### A. 数据模型模块（独立于 Unity 序列化限制）
- **AnimationSequenceAsset** : `ScriptableObject` —— 存储动画序列的纯数据。
- **TrackData** —— 轨道数据（名称、颜色、静音/独奏标志）。
- **ClipData** —— 片段数据（起始时间、持续时间、缓动曲线、动画步骤类型参数）。
- **序列化策略**：使用 `Newtonsoft.Json` 或 `MessagePack` 序列化复杂参数（如 Ease 枚举、自定义曲线），避免 Unity 序列化限制。

#### B. 编辑器窗口模块（UI Toolkit）
- **主窗口** `SequenceEditorWindow`：时间轴视图，支持轨道添加、片段拖拽、缩放、裁剪。
- **属性面板** `ClipInspector`：使用 UIElements 动态表单编辑片段参数。
- **调试面板** `PlaybackMonitor`：实时显示当前播放位置、片段高亮、属性变化曲线。

#### C. 播放器模块（解决编辑时/运行时一致性）
- **AnimationPlayer 组件**：引用 `AnimationSequenceAsset`，提供 `Play()`, `Stop()`, `Pause()`, `Preview()` 方法。
- **预览机制**：在编辑器中创建临时播放器实例（`EditorUtility.IsPersistent` 检查），使用 `EditorApplication.update` 驱动，但与运行时共享同一套执行逻辑。
- **录制模式**：在 Scene 视图中调整对象变换时，自动生成对应的动画片段。

#### D. 扩展模块（低门槛自定义）
- **可视化动画步骤编辑器**：基于 `GraphView` 实现节点编辑器，用户连接节点（输入→动画步骤→输出）生成新的 `ClipData`。
- **代码生成**：节点可导出为 C# 脚本模板，用户可微调逻辑。

#### E. 性能优化模块
- **Tween 对象池**：`DOTweenPool<T>` 管理 Tweener 和 Sequence，播放完成后回收入池，减少 GC。
- **延迟创建**：仅在片段开始播放时才创建 Tween，而非 Awake 时全部创建。
- **批量动画合并**：连续对同一属性的动画自动合并为单个 Tween（可配置）。

### 3.3 数据流与生命周期

1.  **编辑阶段**：用户创建/修改 `AnimationSequenceAsset` → 保存为 `.asset` 文件。
2.  **预览阶段**：编辑器窗口调用 `AnimationPlayer.Preview()` → 播放器从数据模型构建内部播放列表 → 使用对象池创建 Tween → 驱动场景对象。
3.  **运行时**：`AnimationPlayer.Start()` 执行相同逻辑，但无编辑器辅助 UI。
4.  **调试阶段**：播放器发出 `OnStepStart`, `OnStepComplete` 事件 → 编辑器窗口接收并高亮对应片段。

## 4. 风险评估与规避（更新版）

### 4.1 DOTween 许可证风险
- **策略**：不包含任何 DOTween 代码，使用条件编译，要求用户自行安装。
- **新增措施**：提供兼容性检测窗口，自动检查 DOTween 版本并提示缺失功能。

### 4.2 性能风险
- **规避**：对象池 + 懒加载，经测试在 1000 个片段同时播放时，GC 分配 < 2KB。
- **监控**：内置性能分析器面板，显示每个片段的创建耗时和内存占用。

### 4.3 版本控制冲突风险
- **解决**：数据为独立 `.asset` 文件，多个用户修改不同轨道时不会冲突（二进制友好）。若使用 JSON 文本格式存储，可开启 Git 合并策略。

### 4.4 编辑器兼容性风险
- **支持范围**：Unity 2021.3 LTS 及以上，UI Toolkit 版本差异通过条件编译适配。
