# 第一阶段：核心数据模型与播放器 - 规划文档

## 阶段目标

建立 DOTween 可视化编辑器的基础架构，包括数据模型、播放器、适配器层和对象池。确保数据与对象解耦，运行时与编辑时行为一致。

---

## 一、模块划分

### 1.1 数据模型模块 (Runtime/Data)

| 文件 | 说明 |
|------|------|
| `AnimationSequenceAsset.cs` | ScriptableObject，存储动画序列的纯数据 |
| `TrackData.cs` | 轨道数据（名称、颜色、静音/独奏标志、片段列表） |
| `ClipData.cs` | 片段数据（起始时间、持续时间、缓动曲线、动画参数） |
| `AnimationStepData.cs` | 动画步骤基类，可序列化的参数容器 |
| `EasingData.cs` | 缓动曲线数据（支持内置 Ease 和自定义 AnimationCurve） |

### 1.2 动画步骤定义模块 (Runtime/Steps)

| 文件 | 说明 |
|------|------|
| `IAnimationStep.cs` | 动画步骤接口 |
| `AnimationStepBase.cs` | 动画步骤基类（公共逻辑） |
| `TransformStep.cs` | Transform 动画步骤（Move/Rotation/Scale） |
| `PropertyStep.cs` | 通用属性动画步骤 |
| `CallbackStep.cs` | 回调步骤（延迟调用） |

### 1.3 播放器模块 (Runtime/Player)

| 文件 | 说明 |
|------|------|
| `AnimationPlayer.cs` | MonoBehaviour 播放器组件 |
| `SequencePlayer.cs` | 序列播放逻辑（非 MonoBehaviour） |
| `ClipPlayer.cs` | 单个片段播放逻辑 |
| `PlaybackState.cs` | 播放状态枚举与状态机 |

### 1.4 适配器模块 (Runtime/Adapter)

| 文件 | 说明 |
|------|------|
| `IDOTweenAdapter.cs` | DOTween 适配器接口 |
| `DOTweenAdapter.cs` | DOTween 适配器实现 |
| `DOTweenCompatibility.cs` | Free/Pro 版本兼容性检测 |

### 1.5 对象池模块 (Runtime/Pool)

| 文件 | 说明 |
|------|------|
| `TweenPool.cs` | Tweener 对象池 |
| `SequencePool.cs` | Sequence 对象池 |
| `PoolConfig.cs` | 对象池配置 |

## 二、数据结构设计

### 2.1 AnimationSequenceAsset

```csharp
[CreateAssetMenu(fileName = "NewAnimationSequence", menuName = "DOTween Visual/Animation Sequence")]
public class AnimationSequenceAsset : ScriptableObject
{
    public string SequenceName;
    public float TotalDuration;
    public List<TrackData> Tracks = new();
    public LoopType LoopType = LoopType.Restart;
    public int Loops = 1;
}
```

### 2.2 TrackData

```csharp
[Serializable]
public class TrackData
{
    public string TrackName;
    public Color TrackColor = Color.white;
    public bool IsMuted;
    public bool IsSolo;
    public List<ClipData> Clips = new();
}
```

### 2.3 ClipData

```csharp
[Serializable]
public class ClipData
{
    public string ClipName;
    public float StartTime;
    public float Duration;
    public EasingData Easing;
    public AnimationStepData StepData; // 多态存储具体动画参数
}
```

### 2.4 动画步骤多态序列化问题

**问题**：Unity 无法直接序列化多态类型。

**方案选择**（需确认）：

| 方案 | 优点 | 缺点 |
|------|------|------|
| A. ScriptableObject 引用 | Unity 原生支持 | 每个 Step 都是独立文件，管理复杂 |
| B. 序列化字段 + 类型标记 | 简单直接 | 需要自定义 Inspector，字段多时冗余 |
| C. JSON/MessagePack 嵌套 | 完整多态支持 | 需要额外序列化库依赖 |

**建议**：方案 B（序列化字段 + 类型标记），无额外依赖，实现简单。

## 三、核心类交互流程

```
┌──────────────────────────────────────────────────────┐
│                  AnimationSequenceAsset              │
│  (ScriptableObject，存储数据)                        │
└────────────────────┬─────────────────────────────────┘
                     │ 引用
                     ▼
┌──────────────────────────────────────────────────────┐
│                  AnimationPlayer                     │
│  (MonoBehaviour，挂在 GameObject 上)                 │
│  ┌──────────────────────────────────────────────┐  │
│  │           SequencePlayer                      │  │
│  │  - 管理所有 TrackPlayer                       │  │
│  │  - 控制播放/暂停/停止                         │  │
│  └──────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────┘
                     │ 创建并缓存
                     ▼
┌──────────────────────────────────────────────────────┐
│                  DOTweenAdapter                      │
│  - 封装 DOTween API                                 │
│  - 统一 Free/Pro 接口                               │
│  - 从 TweenPool 获取 Tweener                        │
└──────────────────────────────────────────────────────┘
                     │ 使用
                     ▼
┌──────────────────────────────────────────────────────┐
│                  TweenPool                           │
│  - 懒加载创建 Tweener                               │
│  - 播放完成后回收                                    │
└──────────────────────────────────────────────────────┘
```
